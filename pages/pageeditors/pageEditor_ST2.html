Home Page

Technical Description

HTML Techniques

  • <!DOCTYPE html>: Declares the document type and version of HTML, ensuring proper rendering in browsers.
  • <html lang="en">: Specifies the language of the document, which is crucial for screen readers and search engines.
  • <head>: Contains meta-information, such as character set, viewport settings, and links to stylesheets and scripts.
  • <body>: Contains the main content of the HTML document.
  • <header>: Represents the introductory content or navigation links, typically used for the site's header.
  • <section>: Defines a section of the document, such as a hero section, mission section, or initiatives section.
  • <div>: A generic container for grouping and styling elements.
  • <a>: Defines a hyperlink, used for navigation within the site or to external resources.
  • <img>: Embeds an image into the document, enhancing visual content.
  • <svg>: Used to define vector graphics, providing scalable and resolution-independent images.
  • <footer>: Represents content that is typically found at the bottom of the page, such as copyright information and links to related content.

CSS Techniques

  • :root: Defines custom properties (CSS variables) that can be used throughout the stylesheet.
  • html: Sets global styles for the document, such as scroll behavior.
  • body: Styles the document body, including font family and cursor styles.
  • a and button: Styles hyperlinks and buttons, including hover effects to enhance interactivity.
  • header and footer: Styles the header and footer sections, including background color, text color, and layout.
  • flexbox: Used extensively for layout, providing flexible and responsive design.
  • background-image and background-size: Used to set background images and ensure they cover the entire element.
  • position and z-index: Controls the positioning and stacking order of elements.
  • border-radius: Rounds the corners of an element, creating a more visually appealing design.
  • box-shadow: Adds shadow effects to elements, enhancing depth and visual appeal.
  • transition: Smoothly transitions between property values, improving user experience with animations.
  • ::before and ::after: Pseudo-elements used to insert content before or after an element's content.
  • @media queries: Allows styles to adapt to different screen sizes, ensuring responsive design.

Accessibility

How Accessibility Was Addressed:

  1. Semantic HTML: The use of semantic HTML tags (<header>, <section>, <footer>) helps screen readers and other assistive technologies understand the structure of the page.
  2. Alt Text for Images: All images have alt attributes, providing descriptions for screen readers.
  3. Keyboard Navigation: Interactive elements like buttons and links are styled to be visually distinct when focused, ensuring they are usable via keyboard navigation.
  4. Color Contrast: The color scheme ensures sufficient contrast between text and background, making content readable for users with visual impairments.
  5. Responsive Design: The use of viewport meta tags and responsive CSS ensures the page is usable on a variety of devices, including those with smaller screens.
  6. Dynamic Content: JavaScript is used to enhance interactivity but does not rely on it for critical content, ensuring the page remains functional even if JavaScript is disabled.

Table Page

Technical Description

The HTML and CSS techniques used in this page are designed to create a visually appealing and functional table layout that displays information about marine life conservation status. The key HTML tags and CSS properties used are as follows:

  • HTML Table Tags:
    • <table>: Defines the table structure, wrapping all rows and cells.
    • <tr>: Defines a row in the table. Used for both header and data rows.
    • <th>: Defines a header cell in the table. Used for column headers.
    • <td>: Defines a standard cell in the table. Used for data entries.
    • colspan: Attribute used to span a cell across multiple columns.
  • CSS Properties:
    • border-collapse: Ensures that table borders are merged into a single border.
    • width and min-width: Controls the width of the table and ensures it does not become too narrow.
    • border-radius and overflow: Creates rounded corners for the table and handles overflow content.
    • background-color: Sets the background color of table rows and cells for visual distinction.
    • text-align: Aligns the text within table cells for better readability.
    • height: Sets the height of table rows for consistent spacing.

These tags and properties ensure that the table is both functional and visually appealing, while also considering accessibility for all users.

Accessibility

Accessibility was addressed through several techniques:

  • Semantic HTML: Using <table>, <tr>, <th>, and <td> tags ensures that the table structure is clear and understandable to screen readers.
  • Header Cells: Using <th> for header cells helps screen readers identify the column headers, making it easier for users to understand the data.
  • Text Alternatives: Images within table cells have alt attributes, providing descriptions for screen readers.
  • Color Contrast: Sufficient color contrast between text and background ensures readability for users with visual impairments.
