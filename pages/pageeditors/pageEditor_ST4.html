Profile Page

Technical Description

The profile page is structured using div tags to separate different sections, making it easier to apply styling. The profile section, which includes the profile image, cover image, and related content, is styled using display: flex with justify-content: center and a column flex direction to align elements effectively. Social media icons are implemented using Scalable Vector Graphics (SVG) for better scalability. The "About" and "Who am I" sections are placed side by side within a common container using flex display, a technique also applied to the skills and experience sections. The data in the about section is added using a table structure. Section dividers are created using the hr tag, while the progress bar effect is achieved through a div with custom styling. Links in the experience section are styled with :hover effects for interactivity. Different UN goals are presented in individual cards containing links to their respective pages. The volunteering availability section includes a calendar, where the span tag is used to apply styling within a div. The calendar layout is structured using grid-template-columns to maintain uniform spacing. A "Contact Me" button triggers a modal form using JavaScript, allowing users to input data with relevant form field types. Hover effects are applied to content sections and buttons to enhance visual appeal, and a background image is added for a more engaging design.

Accessibility

  • I used meaningful tags like header, section, h2, and p instead of just div to enhance screen reader navigation.
  • All images, including the profile and cover images, have alt attributes to provide descriptive text for users relying on screen readers.
  • Interactive elements like buttons and links are styled appropriately and can be navigated using the Tab key, improving usability for those who don't use a mouse.
  • The text colors and backgrounds maintain sufficient contrast to be readable for users with visual impairments.
  • I implemented :hover styles on buttons and links to improve visibility when users navigate.
  • The page uses percentages for layout elements to support zooming and different screen sizes without breaking the design
  • Users are bale to easily access the contact modal through a click of a button and exit the modal by either clicking the black space around the form or the close button.
  • The background opacity is reduced when the modal is opened in order to give the users' full attention to the modal.

Sitemap Page

Technical Description

The sitemap page is structured using an svg based layout to visually represent the website's structure, ensuring clarity and ease of navigation. The page is styled with a full-width background image for a visually engaging experience. The central title is positioned using text-align: center and a large font size for emphasis. The sitemap itself is constructed with SVG elements, where each main section is represented as a rectangular node (rect class="main-node-svg"), styled with a light blue fill and a transition effect for hover interactions. Sub-sections are created using (rect class="sub_node"), maintaining a consistent design with white-filled rectangles and smooth hover transitions. Connecting lines (line class="line") are drawn to visually link parent and child nodes, enhancing the hierarchical structure. Text labels (text) are centered within each node using text-anchor: middle and dominant-baseline: middle for proper alignment. Interactive elements are made clickable through onclick functions that trigger JavaScript navigation. Additionally, hyperlinks (a tags) allow external access to relevant pages. The grid-based alignment of elements is controlled using transform matrices, ensuring a structured and evenly spaced display. Hover effects (:hover) on nodes and sub-nodes provide visual feedback by changing colors and strokes when hovered. The page maintains accessibility features, such as tabindex="0" for keyboard navigation and color contrasts for readability. The overall design ensures a user-friendly and interactive visualization of the website's structure while maintaining responsiveness and ease of navigation.

Accessibility

  • I used meaningful tags like header, section, h2, and p instead of just div to enhance screen reader navigation.
  • Interactive elements such as nodes (rect elements) and links are assigned tabindex="0", ensuring users can navigate the sitemap using the Tab key without requiring a mouse.
  • The SVG-based sitemap includes text elements (text) with clear, readable labels, improving accessibility for screen reader users.
  • I implemented :hover styles on buttons and links to improve visibility when users navigate.
  • The page uses percentages for layout elements to support zooming and different screen sizes without breaking the design

Content Page

Technical Description

The content page is structured using section and tabindex="0" tags for organization. The hero section features a background image with a gradient overlay (::before) and flexbox for centering the title and button. The introduction section uses a solid background color, centered text, and padding for readability. The main content is divided into a sidebar (sticky navigation with hover effects) and a main content area (flexbox layout for info boxes with images and text). The challenges section mirrors the introduction's styling, with bullet points for challenges. The conclusion section repeats the introduction's design, emphasizing the call to action. The "Other Contents" section uses a grid layout for related content cards, with hover effects and image overlays for interactivity. A "Back to Top" button is fixed at the bottom right, with a hover effect for scaling. The page employs flexbox, grid, and absolute positioning for a responsive layout, with gradients, shadows, and SVG icons for a cohesive and engaging design.

Accessibility

  • I used meaningful tags like header, section, h2, and p instead of just div to enhance screen reader navigation.
  • All images, including those in the hero section and info boxes, have descriptive alt attributes, providing context for users relying on screen readers or those with images disabled.
  • I implemented :hover styles on buttons and links to improve visibility when users navigate.
  • The page uses percentages for layout elements to support zooming and different screen sizes without breaking the design
  • The sidebar with quick navigation links is sticky, ensuring it remains accessible as users scroll through the page.
  • Headings (h1, h2, etc.) are used hierarchically to structure the content, making it easier for screen readers to navigate and understand the page's organization.

Challenges and Lessons Learned

Challenge: Ensuring all interactive elements (buttons, links, forms) were fully navigable via the Tab key required testing and fixing issues like focus traps or missing focus indicators.

Lesson Learned: Implementing keyboard navigation from the start and testing it regularly can help identify and address accessibility issues early in the development process.


Challenge: Ensuring the svg elements (rect, line) were positioned at the correct positions.

Lesson Learned: Using the correct x and y coordinates and ensuring the correct width and height of the elements are set can help in positioning the elements correctly.


Challenge: Implementing accessibility features required additional time and resources, which were sometimes limited.

Lesson Learned: Prioratizing major accessibility features at the beginning of the project and implementing others later.


Challenge: Collaborating with team members.

Lesson Learned: Using collaboration tools like GitHub and holding regular meetings helped us to maintain a understanding of the progress of the website.

Compliance

Text Alternatives: All images, including the hero section background and info box images, have descriptive alt attributes. This ensures that users relying on screen readers can understand the content of the images.

Adaptable Content: The layout uses flexible units (e.g., percentages, flexbox, and grid) to ensure the content adapts to different screen sizes and zoom levels without breaking the design.

Contrast and Readability: Text and background colors maintain sufficient contrast (e.g., white text on dark backgrounds) to meet WCAG 2.1 standards, ensuring readability for users with visual impairments.

Structure: Proper semantic HTML structure for better readability and navigation.

Ethical Aspect: No prohibited, harmful, or misleading information is presented, aligning with ethical content standards
